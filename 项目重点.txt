1. 工具类
MatPipeline
背景：OpenCV Java 官方绑定没有提供自动内存管理，需要开发者手动释放。
原因：自动管理会引入额外开销，如果每次都要注册到 GC，性能会下降
官方文档摘录：
Memory management:
All Mat objects should be explicitly released using Mat.release() method when they are no longer needed. Although finalize() method will eventually release the native memory, it's not guaranteed to be called in a timely manner and may lead to out-of-memory errors in long-running applications.
内存管理：
所有 Mat 对象在不再需要时都应该显式使用 Mat.release() 方法释放。虽然 finalize() 方法最终会释放原生内存，但不能保证及时调用，可能导致长时间运行的应用出现内存溢出错误。

方案：DSL风格+自动管理
特性：
	链式调用：DSL 风格，代码简洁易读
	自动释放中间资源：所有中间 Mat 自动释放，无需手动管理
	灵活的结果获取：支持 get() / getAll() / use()
	丰富的图像操作：涵盖 OpenCV 常用图像处理功能
解决的问题：
	中间 Mat 释放：手动管理，易遗漏
	代码可读性：繁琐的 try-finally
	finalize 不可靠：依赖不可靠机制

适合场景：OCR 预处理、图像处理流水线、需要频繁创建临时 Mat 的应用
不适合场景：需要极精细控制每个 Mat 生命周期、与现有 OpenCV 代码大量混用的项目
